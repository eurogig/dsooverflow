## Setup

### Local Development

1. Clone the repository:
   ```bash
   git clone <your-repo-url>
   cd dsooverflow
   ```

2. Set your API key in `config.js`:
   ```javascript
   window.API_CONFIG = {
       API_KEY: 'your-api-key-here'
   };
   ```

3. Serve the files using a local server:
   ```bash
   # Using Python
   python3 -m http.server 8000
   
   # Using Node.js (if you have http-server installed)
   npx http-server
   ```

4. Open `http://localhost:8000` in your browser

### GitHub Pages Deployment

1. **Set up GitHub Secrets:**
   - Go to your repository settings
   - Navigate to Secrets and variables â†’ Actions
   - Add a new secret named `BUZZSPROUT_API_KEY` with your API key value

2. **Enable GitHub Pages:**
   - Go to repository settings
   - Navigate to Pages
   - Under "Source", select "GitHub Actions"

3. **Push to main branch:**
   - The GitHub Actions workflow will automatically deploy your site
   - The workflow will inject the API key from secrets into `config.js` during build

## How It Works

1. **Initial Load:** The site fetches all episodes from the Buzzsprout API and displays the first 2 pages (24 episodes by default).

2. **Infinite Scroll:** As users scroll down, more episodes are automatically loaded using the Intersection Observer API.

3. **API Key Management:** 
   - For local development: Set the API key directly in `config.js`
   - For production: GitHub Actions injects the API key from secrets during deployment

## Customization
